# Cursor Prompt — Wire HandoffAI Frontend to Backend

> Paste everything below the divider into Cursor's chat. Cursor has access to all files in the workspace, so it can edit them directly. This prompt covers both the frontend wiring and a small backend tweak that makes session history work end-to-end.

---

## Task

Wire the HandoffAI frontend (HTML + Babel-in-browser, no build step) to the FastAPI backend. Replace placeholder data with real API calls. Keep certain mocks intentionally — see "What stays mocked" below.

## Backend endpoints available

- `POST /api/decode` — body `{raw_text, nurse_label?, shift_label?}` → `{id, patients: [...], global_glossary, confidence, ambiguities}`. Each patient has `{id, room, name_or_label, sbar, flags, open_loops, abbreviations_used}`.
- `GET /api/patients/by-room/{room}` → array of patient records with `{id, handoff_id, created_at, room, name_or_label, sbar, flags, open_loops, verified, nurse_label, shift_label}`.
- `POST /api/patient-records/{record_id}/verify` → `{verified: true}`.
- `GET /health` → status check.

## What stays mocked (intentional, do not change)

- `window.PATIENTS` array — this is the nurse's assigned patient roster. In production it would come from an EHR integration (Epic/Cerner). Out of scope for the hackathon.
- `window.NURSE` object — represents the logged-in user. No auth in scope.
- `window.SYSTEM_PROMPT` — can be removed since backend owns it now, but harmless to keep.

## What gets replaced

- `window.NVIDIA_API_KEY` and the direct NVIDIA call from `screen2.jsx` — frontend should never have the key.
- `window.MOCK_NOTES` — replaced by `/api/decode` response. Keep as offline fallback only.
- `window.SESSION_HISTORY` — replaced by `/api/patients/by-room/{room}` response. Keep as offline fallback only.

### 2. `screen2.jsx` — RecordingScreen

Replace the entire `handleProcess` function with this version. It calls the backend instead of NVIDIA directly, passes the nurse name for session history attribution, and threads the IDs through to the next screen.

```jsx
const handleProcess = async () => {
  if (recording) {
    recognitionRef.current?.stop();
    setRecording(false);
  }
  setProcessing(true);

  const backendUrl = window.BACKEND_URL || 'http://localhost:8000';

  try {
    const res = await fetch(`${backendUrl}/api/decode`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        raw_text: transcript,
        nurse_label: `${NURSE.name}, ${NURSE.credentials}`,
        shift_label: NURSE.shift,
      }),
    });

    if (!res.ok) {
      const err = await res.json().catch(() => ({ detail: `Backend ${res.status}` }));
      throw new Error(err.detail || `Backend returned ${res.status}`);
    }

    const data = await res.json();
    // Backend returns {id, patients: [...]}. Single-patient dictation returns one entry.
    const patientRecord = data.patients?.[0];
    if (!patientRecord) throw new Error('Backend returned no patient records');

    // Reshape to the structure NotesPreviewScreen expects, plus pass IDs through
    const notes = {
      sbar: patientRecord.sbar,
      flags: patientRecord.flags || [],
      open_loops: patientRecord.open_loops || [],
      abbreviations_used: patientRecord.abbreviations_used || [],
      _handoff_id: data.id,
      _record_id: patientRecord.id,
    };

    setProcessing(false);
    onProcessed(notes, transcript);
  } catch (err) {
    console.error('Backend decode failed, falling back to mock:', err);
    setProcessing(false);
    // Offline fallback so the demo doesn't die if backend is down
    const fallback = MOCK_NOTES[patient.id] || MOCK_NOTES['p1'];
    onProcessed({ ...fallback, _handoff_id: null, _record_id: null }, transcript);
  }
};
```

The `_handoff_id` and `_record_id` fields are stashed inside the notes object so they thread through `app.jsx` state without changing the function signatures of `onProcessed`/`onApprove`.

### 4. `screen4.jsx` — PatientInfoScreen

This is the biggest change. Replace the line `const history = SESSION_HISTORY[patient.id] || [];` with a stateful fetch.

At the top of the `PatientInfoScreen` function, replace:
```jsx

```

With:
```jsx
const [history, setHistory] = React.useState([]);
const [historyLoading, setHistoryLoading] = React.useState(true);
const currentRecordId = approvedNotes?._record_id;

React.useEffect(() => {
  const backendUrl = window.BACKEND_URL || 'http://localhost:8000';
  let cancelled = false;

  const loadHistory = async () => {
    try {
      const res = await fetch(`${backendUrl}/api/patients/by-room/${encodeURIComponent(patient.room)}?limit=10`);
      if (!res.ok) throw new Error(`Backend ${res.status}`);
      const records = await res.json();
      if (cancelled) return;

      // Filter out the current record (it's shown separately as "This session")
      const others = records.filter(r => r.id !== currentRecordId);

      // Map backend shape -> SessionCard prop shape
      const mapped = others.map(r => ({
        id: r.id,
        nurse: r.nurse_label || 'Previous shift',
        nurse_id: '—',
        timestamp: new Date(r.created_at).toLocaleString([], {
          month: 'short', day: 'numeric', year: 'numeric',
          hour: '2-digit', minute: '2-digit',
        }),
        shift: r.shift_label || _deriveShiftFromTime(r.created_at),
        flags: r.flags || [],
        summary: r.sbar?.situation || '',
      }));

      setHistory(mapped);
    } catch (err) {
      console.warn('History fetch failed, falling back to mock:', err);
      if (!cancelled) setHistory(SESSION_HISTORY[patient.id] || []);
    } finally {
      if (!cancelled) setHistoryLoading(false);
    }
  };

  loadHistory();
  return () => { cancelled = true; };
}, [patient.room, currentRecordId]);

// Derive a shift label from timestamp if backend didn't provide one
function _deriveShiftFromTime(iso) {
  const h = new Date(iso).getHours();
  if (h >= 7 && h < 19) return 'Day shift';
  return 'Night shift';
}
```

async def save_handoff(
    raw_input: str,
    response: dict,
    nurse_label: Optional[str] = None,
    shift_label: Optional[str] = None,
) -> str:
    pool = _pool_or_raise()
    async with pool.acquire() as conn:
        async with conn.transaction():
            handoff_id = await conn.fetchval(
                """
                INSERT INTO handoffs (raw_input, structured_output, confidence, nurse_label, shift_label)
                VALUES ($1, $2::jsonb, $3, $4, $5)
                RETURNING id
                """,
                raw_input,
                json.dumps(response),
                response.get("confidence", "medium"),
                nurse_label,
                shift_label,
            )

### `db.py` — `get_patient_history_by_room` joins handoffs

Update the query to pull nurse/shift labels from the parent handoff:

```python
async def get_patient_history_by_room(room: str, limit: int = 10) -> list[dict]:
    pool = _pool_or_raise()
    async with pool.acquire() as conn:
        rows = await conn.fetch(
            """
            SELECT pr.id, pr.handoff_id, pr.created_at, pr.room, pr.name_or_label,
                   pr.sbar, pr.flags, pr.open_loops, pr.verified,
                   h.nurse_label, h.shift_label
            FROM patient_records pr
            LEFT JOIN handoffs h ON h.id = pr.handoff_id
            WHERE pr.room = $1
            ORDER BY pr.created_at DESC
            LIMIT $2
            """,
            room,
            limit,
        )
        return [
            {
                "id": str(r["id"]),
                "handoff_id": str(r["handoff_id"]),
                "created_at": r["created_at"].isoformat(),
                "room": r["room"],
                "name_or_label": r["name_or_label"],
                "sbar": json.loads(r["sbar"]),
                "flags": json.loads(r["flags"]),
                "open_loops": json.loads(r["open_loops"]),
                "verified": r["verified"],
                "nurse_label": r["nurse_label"],
                "shift_label": r["shift_label"],
            }
            for r in rows
        ]
```

## Common things that go wrong

- **CORS errors in browser console.** Make sure the backend's `ALLOWED_ORIGINS` env var includes `http://localhost:5173`. Already there if you used the cheat sheet.
- **`data.patients?.[0]` is undefined.** Backend returned an empty patients array — usually means Nemotron returned valid JSON with `patients: []`. Check the backend logs for the raw model output. Tweak the system prompt to be more insistent that single-patient dictation should still produce one patient entry.
- **Verify call returns 404.** The `_record_id` got lost between screens. Check that `editedNotes._record_id` exists in `screen3.jsx` — the `JSON.parse(JSON.stringify(notes))` clone should preserve it.
- **Session history empty even after multiple decodes.** Check that the backend actually persisted them: hit `/api/handoffs?limit=5` and look for rows. If they exist, the room field on patient_records may not match. Add a `console.log(patient.room)` in screen4 and compare with what's in Supabase Table Editor.
- **`nurse_label` always null in history.** The backend tweaks didn't get applied. Check `models.py` for the field, restart uvicorn (auto-reload sometimes misses Pydantic schema changes).
